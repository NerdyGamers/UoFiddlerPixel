Integrate animations into Anim.mul.

This tutorial explains how to insert new animations into the Anim(X).mul file.

Step 1: Select the 'Animations' tab.

First, we select the 'Animations' tab. Then, we choose 'Settings' > 'Animation Edit' from the bottom.

Step 2: Open the Animation Editor.

Subsequently, the Animation Editor opens. There, under 'Choose anim File', we select the respective Animations .mul file that we want to use to insert new animations.

Step 3: Search for ID for the New Animation

Now we search for the ID where we want to import the new animations. One can also initiate a search to find free IDs and then choose a spot.

Step 4: Import New Animation

Now we select the animation from the directory and import it.

Step 5: Animation Imported

The animation has now been imported into the ID slot.

Step 6: Save

Now, save the respective Anim.mul and navigate to the directory where the data will be stored. Afterwards, copy it into the Ultima Online directory.

Step 7: Settings for Bodyconv.def and mobtypes.txt

Now, adjustments are made in the 'Bodyconv.def' and 'mobtypes.txt' files in the Ultima Online directory. First, check if the same numbers are not already present, and then enter the settings for the respective animation you want to address. As an example, I've taken number 1000 and 1001 for the elephant and 1000 for the camel.

Step 8: Reload UoFiddler

After entering and saving the settings in the 'Bodyconv.def' and 'mobtypes.txt' files, you can restart UO Fiddler and check if the animations are listed, also with the correct address, which we will then use for the server script. Likewise, we update the 'AnimationList.xml' to correctly display the animation at startup.

Step 9: Reload UoFiddler

Now, one can check again to see if everything is working properly.

Step 10: AnimationList.xml

If desired, one can directly add their entry of the new animation to the 'AnimationList.xml' so it is listed at the start of UO Fiddler. To do this, navigate to your 'AppData\Roaming\UoFiddler' path and edit the file. Give the animation a name and enter the same body address, for example, 1000, as specified in the example.

Step 11: Create Script for the Server

Next, create a script for the server, specify the body number there, and then start the server.
